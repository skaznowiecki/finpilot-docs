## Gestión de Usuarios, Roles y Permisos

FinPilot incluye un sistema completo de gestión de usuarios, roles y permisos que te permite controlar quién puede acceder a la plataforma y qué acciones puede realizar cada usuario.

## ¿Qué es el sistema de usuarios, roles y permisos?

El sistema de gestión de usuarios, roles y permisos de FinPilot te permite:

- **Gestionar usuarios**: Crear, editar, activar y desactivar usuarios de tu empresa
- **Asignar roles**: Controlar el acceso mediante roles predefinidos o personalizados
- **Controlar permisos**: Definir qué acciones puede realizar cada usuario de forma granular

## Flujo de trabajo típico

El proceso típico para configurar usuarios en FinPilot es:

1. **Crear o seleccionar un rol**: Puedes usar uno de los roles predefinidos del sistema (admin, read, write) o crear un rol personalizado con permisos específicos
2. **Crear el usuario**: Agrega un nuevo usuario directamente en el sistema con sus credenciales
3. **Asignar el rol**: Asigna el rol apropiado al usuario para que tenga los permisos correctos
4. **Acceso inmediato**: El usuario puede acceder inmediatamente con las credenciales creadas

## Roles del sistema

FinPilot incluye tres roles predefinidos que cubren la mayoría de casos de uso:

- **admin**: Acceso completo a todos los recursos y gestión de usuarios (33 permisos)
- **read**: Acceso de solo lectura a todos los recursos (10 permisos)
- **write**: Acceso de lectura y escritura a todos los recursos excepto gestión de usuarios (26 permisos)

También puedes crear **roles personalizados** con combinaciones específicas de permisos según las necesidades de tu organización.

## Casos de uso comunes

**Equipo pequeño**
- Un usuario con rol **admin** para el administrador principal
- Usuarios con rol **write** para el equipo que gestiona facturas

**Equipo grande con departamentos**
- Usuarios con rol **admin** para administradores
- Roles personalizados para cada departamento (contabilidad, compras, etc.)
- Usuarios con rol **read** para consultas y reportes

**Control estricto**
- Roles personalizados con permisos muy específicos
- Separación clara entre usuarios que pueden gestionar facturas y usuarios que solo pueden verlas

<Tip>
**Consejo**: Comienza usando los roles predefinidos del sistema. Puedes crear roles personalizados más adelante cuando tengas necesidades específicas.
</Tip>
